# 多模型蒸馏投喂框架
> 版本: v1.0 | 更新: {{DATE}}
> ⚠️ 核心原则: 分散投喂、独立输出、人工筛选、统一规则

---

## 一、模型分工架构

### 1.1 国内在线主力模型（日常调用）
| 模型 | 核心用途 | 优势 | 限制 |
|------|----------|------|------|
| **Kimi (月之暗面)** | 长文本逻辑梳理、复盘归档、规则总结 | 超长上下文、中文理解强 | 量化推理一般 |
| **DeepSeek** | 量化逻辑推理、代码生成、数据分析 | 数学能力强、推理深 | 长文本不如Kimi |
| **通义千问** | 信息提取、文档分析、规则验证 | 多模态、稳定 | 幻觉率较高 |
| **豆包(字节)** | 快速研判、数据总结、新闻解读 | 响应快、信息面广 | 深度不足 |

### 1.2 海外离线辅助模型（仅蒸馏用）
| 模型 | 核心用途 | 调用规则 |
|------|----------|----------|
| **GPT-4/4o** | 深度逻辑拆解、规则漏洞分析、反向推演 | 禁止高频交易调用，防封号 |
| **Claude 3.5** | 长文复盘、复杂推理、风险识别 | 仅用于错误案例复盘 |

## 二、蒸馏投喂执行流程

### 步骤1: 分散投喂
将同一套交易规则/案例，分别投喂给6个模型：

```
投喂内容模板:
"请基于以下交易规则/市场案例，输出:
1. 你认为的核心判断逻辑是什么?
2. 该规则的适用条件和边界?
3. 可能存在哪些漏洞或风险?
4. 给出一个反例
5. 如果规则有误，正确的逻辑应该是什么?"

[在此处粘贴你的规则/案例]
```

### 步骤2: 独立输出记录表
| 模型 | 核心逻辑 | 适用条件 | 漏洞/风险 | 反例 | 修正建议 |
|------|----------|----------|-----------|------|----------|
| Kimi | | | | | |
| DeepSeek | | | | | |
| 通义千问 | | | | | |
| 豆包 | | | | | |
| GPT-4 | | | | | |
| Claude | | | | | |

### 步骤3: 人工汇总筛选
- [ ] 标记6个模型中均提到的逻辑（高置信度）
- [ ] 标记5个模型提到的逻辑（需验证）
- [ ] 标记单一模型特有观点（可能是幻觉或独特洞察）
- [ ] 标注明显错误的输出（模型幻觉）

### 步骤4: 规则迭代
- [ ] 将高置信度逻辑写入规则手册
- [ ] 将独特观点作为观察项记录
- [ ] 建立"模型错误黑名单"

### 步骤5: 沉淀标准
- [ ] 输出新版规则（仅包含筛选后的逻辑）
- [ ] 标记来源（哪些模型贡献了该规则）
- [ ] 记录迭代版本

## 三、模型错误黑名单

| 模型 | 固定偏见/缺点 | 适用场景规避 |
|------|--------------|-------------|
| GPT-4 | 偏向西方视角、对中国政策解读偏颇 | 国内政策分析慎用 |
| Claude | 过度激进保守、对量化数据不敏感 | 仓位算法不依赖 |
| 通义千问 | 容易给出"正确的废话"、缺乏深度 | 需要人工二次确认 |
| Kimi | 偏好长推理、可能过度复杂化 | 简单判断不需要 |
| DeepSeek | 过度数学化、忽视情绪面分析 | 情绪决策补充 |
| 豆包 | 信息广度够但深度不足 | 仅用于信息收集 |

## 四、板块Agent蒸馏模板

### 4.1 外贸Agent蒸馏
```
投喂模型: DeepSeek + Claude + Kimi
输入信息:
- 最新港口吞吐量数据
- 集装箱运价指数(SCFI)
- BDI波罗的海指数
- 跨境支付结算数据
- 自贸区政策文件

输出要求:
1. 当前出口处于拐点哪个位置?(底部/上升/顶部/下降)
2. 数据拐点→产能拐点的传导时间预估
3. 哪些制造业最先受益?
4. 风险因素有哪些?
5. 给出景气度评分(1-10)
```

### 4.2 电力周期Agent蒸馏
```
投喂模型: 通义千问 + DeepSeek + 豆包
输入信息:
- 全社会用电量
- 工业用电细分数据
- 高耗能行业开工率
- 新能源消纳数据
- 电网负荷峰值

输出要求:
1. 当前电力周期位置(低谷/回升/高峰/回落)
2. 哪些高耗能行业受到供给约束?
3. 预计缺口/过剩何时出现?
4. 产业链传导路径
5. 相关标的选择建议
```

### 4.3 科技卡脖子Agent蒸馏
```
投喂模型: Kimi + GPT-4 + DeepSeek
输入信息:
- 半导体设备进口数据
- 先进制程突破公告
- 国产替代政策
- AI算力需求数据
- 低空经济政策

输出要求:
1. 当前处于弯道超车哪个阶段?
2. 哪些环节已经突破?哪些仍为瓶颈?
3. 资金炒作处于哪个阶段?(预期/验证/兑现)
4. 未来30天可能的事件催化
5. 风险因素
```

### 4.4 天气周期Agent蒸馏
```
投喂模型: 豆包 + 通义千问 + Claude
输入信息:
- 台风路径预报
- 极端气温预报
- 港口停工预警
- 厄尔尼诺/拉尼娜指数
- 电力负荷预测

输出要求:
1. 未来1-7天天气事件影响评估
2. 受影响港口/产业链
3. 供需错配的可能性
4. 季节性交易机会
5. 风险预警级别
```

## 五、每日AI研判执行清单

### 5.1 开盘前(8:00-9:15)
- [ ] 收集隔夜外盘数据(美股/商品/汇率)
- [ ] 获取当日财经要闻
- [ ] 天气Agent输出港口影响
- [ ] 生成《今日市场全景预判》(豆包)

### 5.2 盘中(9:30-15:00)
- [ ] 监控资金流向异动
- [ ] 板块联动观测(DeepSeek快速分析)
- [ ] 风险事件实时预警

### 5.3 收盘后(15:00-16:00)
- [ ] 数据归档至素材库
- [ ] 当日交易盈亏统计
- [ ] 信号胜率更新
- [ ] 规则执行率检查

### 5.4 盘后深度复盘(周日晚)
- [ ] 多模型联合复盘(各模型独立输出)
- [ ] 逻辑漏洞修正
- [ ] 规则手册迭代
- [ ] 模型错误黑名单更新
